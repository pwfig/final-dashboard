About CS498 Data Visualization

By

Paul Figliozzi

INTRODUCTION

This narrative visualization is regarding the nature and content of the course CS498 Data Visualization (Summer, 2017).

This subject was chosen as a study investigating the type of information which can be gleaned given the use of data which is primarily hierarchical in nature.

It uses a custom dashboard interface built using HTML5, JavaScript and the D3 library.

NARRATIVE STRUCTURE

The narrative is structured as a hybrid "martini glass", where:

GUIDED PRESENTATION (Slideshow)

The presentation represents the "stem" of the martini glass. Upon invocation (startup), the reader is directed to follow the slideshows in a linear fashion by progressing through the slides using the provided navigation controls.

The reader can track progress through the slides using the progress indicator. The reader can also (later) review the slides in a non-linear fashion by jumping to a slide via clicking on the appropriate progress indicator.

ANALYSIS & DISCOVERY (Charts and Graphs)

This represents the "cup" portion of the martini glass, where the reader can directly navigate to the charts via button selection from the left-side menu or by clicking on the chart (right side).

SCENES

A storyboard was used to construct the narrative, resulting in the resulting scenes:

FIRST SCENE

The dashboard interface itself represents the first scene encountered.

SLIDESHOW SCENES

Each slide within the slide show is a scene. These scenes are progressive in nature in telling the story of the narrative visualization.

CHART SCENES

Each chart presents information supporting analytical results and conclusions mentioned in the slide, or the ability for the viewer to explore the data (self-guided discovery) of the data.

ANNOTATIONS

Tooltips are used to highlight information in the charts and area location in the user interface.

Comments and descriptions are shown in the comment area (middle-lower area) to describe highlights regarding content shown in the main presentation area.

PARAMETERS

CURRENT DASHBOARD AREA

Variable to track which dashboard is currently selected for use (about, slideshow, chart types).

AREA TRANSITION STATE

Tracks whether the area is in a zoomed or un-zoomed state.

HIERARCHICAL CHART NAVIGATION

Variable to track current position selected within the hierarchy.

CURRENT SLIDESHOW POSITION

Variable tracks current, previous and next slides within the slideshow.

COMMENT AREA

Variables to track state of the comment area (clear, populated) and contents shown, comments associated with other areas for display (lookup table).

TRIGGERS

User interface events used in this narrative include:

BUTTON TRIGGERS/EVENTS

MAIN MENU NAVIGATION

This vertical button group provides self-guided dashboard navigation:

BUTTON SELECT

Triggers a transition to the selected dashboard area (About, Slideshow, Chart Type) or action (Exit).

BUTTON HIGHLIGHT(Orange)

CONSTANT (After BUTTON SELECT) highlight shows current dashboard area of focus,

TEMPORARY (Cursor FOCUS Event) highlight shows area to be selected if pressed.

SLIDESHOW NAVIGATION

Slide Buttons provide slideshow navigation:

FORWARD/PREVIOUS - Selection triggers guided (linear) slide movement,

PROGRESS BAR

BAR SELECTION (After BUTTON SELECT) - Triggers a jump to the matching slide (self-guided, non-linear slide movement).

BAR HIGHLIGHT (After BUTTON SELECT)- Bar within the Progress Bar highlights relative slide position within the slideshow.

CURSOR POSITIONING TRIGGERS

BUTTON FOCUS - Moving the mouse over an area of focus (button) shows button selection.

DASHBOARD AREA HOVER - Hovering the mouse over a dashboard area shows a tooltip indicating the area name. Area borders may also highlight.

CHART AREA HOVER - May show a tooltip annotation of interest within the chart (varies by chart type).

AREA TRIGGERS

ENTER/EXIT - Transitioning to an area may trigger highlighting, chart/data load/re-load, or other transitions (zooming/un-zooming).

COMMENT AREA - Slide transitions or selection of chart areas may cause the comment area to load comments, descriptions or annotations of interest related to the area of focus.
